# AI 기반 개발 규칙

이 문서는 AI가 웹사이트의 코드 수정 및 개선 작업을 수행할 때 따라야 할 핵심 규칙들을 정의합니다. 모든 변경 사항은 아래 규칙에 따라 일관성 있게 적용되어야 합니다.

## 1. 모바일 우선 원칙 (Mobile-First Principle)

- **정의**: 모든 UI/UX 디자인과 CSS 스타일링은 모바일 화면을 기준으로 시작해야 합니다.
- **실행 지침**:
    - 가장 작은 화면(예: 360px 너비)에서 디자인이 완벽하게 보이도록 스타일을 작성합니다.
    - 미디어 쿼리(`@media`)를 사용하여 화면이 커짐에 따라(min-width) 레이아웃을 조정하고 요소를 추가합니다.
    - 예: `@media (min-width: 768px) { ... }`

## 2. 상대 단위 사용 규칙 (Relative Units Rule)

- **정의**: 고정된 픽셀(`px`) 단위 대신 상대적인 단위를 사용하여 유연하고 접근성 높은 디자인을 구현합니다.
- **실행 지침**:
    - **폰트 크기**: `rem` 단위를 사용합니다. `html` 요소에 기준 폰트 크기(예: `16px`)를 설정하고, 다른 모든 텍스트 요소는 `rem`으로 지정합니다.
    - **간격 및 레이아웃**: `rem`, `em`, `%`, `vw`, `vh` 단위를 사용합니다.
        - `rem`: 전체적인 레이아웃 간격에 사용합니다.
        - `em`: 특정 요소의 폰트 크기에 비례하는 간격이 필요할 때 사용합니다.
        - `%`, `vw`, `vh`: 컨테이너나 뷰포트에 따라 크기가 변해야 하는 요소(예: 너비, 높이)에 사용합니다.
    - `px`는 `border` 너비와 같이 절대적인 크기가 필요한 예외적인 경우에만 사용합니다.

## 3. BPM 오버레이 및 슬라이더 정렬 규칙

- **정의**: 사용자가 템포를 직관적으로 조절할 수 있도록 BPM 표시부와 컨트롤러의 시각적 정렬을 유지합니다.
- **실행 지침**:
    - **BPM 오버레이**: 화면 정중앙에 위치해야 합니다. Flexbox 또는 Grid를 사용하여 수평 및 수직 중앙 정렬을 구현합니다. (`display: flex; justify-content: center; align-items: center;`)
    - **BPM 슬라이더 및 버튼**:
        - 슬라이더와 양옆의 `+`, `-` 버튼은 수직으로 완벽하게 정렬되어야 합니다.
        - Flexbox의 `align-items: center;` 속성을 사용하여 이들을 감싸는 컨테이너 내부에서 수직 중앙 정렬을 맞춥니다.
